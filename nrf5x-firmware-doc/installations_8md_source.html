1 Steps for setting up the repository {#install_instruction}
2 =======================
3 These instructions are valid for a x64 Linux system with a Debian based environment.
4 \n First you need to setup the working environment and the repositories for editing and compiling the Firmware files.
5 
6 Steps for compiling code and generating the hex files
7 ------
8 
9 - Install ARM GCC Compiler
10  - You will have to install the following version of ARM GCC Compiler:[GNU Arm Embedded Toolchain: 6-2017-q2-update June 28, 2017] (https://developer.arm.com/-/media/Files/downloads/gnu-rm/6-2017q2/gcc-arm-none-eabi-6-2017-q2-update-linux.tar.bz2?revision=2cc92fb5-3e0e-402d-9197-bdfc8224d8a5?product=GNU%20Arm%20Embedded%20Toolchain,64-bit,,Linux,6-2017-q2-update)
11  - Extract the above to location "/usr/local".
12 - Install srecord using:
13 
14  sudo apt-get install srecord
15 
16  \n Srecord's `srec_cat` command is used to merge hex files before flashing to the target device.
17 - Clone the Github repository from https://github.com/Appiko/nrf5x-firmware
18 - Clone the GitLab repository to the same directory as "nrf5x-firmware", from https://gitlab.com/appiko/dfu_nrf_sdk15.git \n Its not available in the public repository as it does not have an open license.
19 
20 Documentation with Doxygen
21 ------
22 We use Doxygen for all our documentation. You can install Doxygen from your package manager or manually from from the official website.
23 
24 http://www.doxygen.nl/download.html
25 
26 
27 To generate documentation Navigate to `nrf5x-firmware/doc` and run
28 
29 `$ doxygen Appiko.doxyfile`
30 
31 The `html` documentation is stored under `./build/html/`
32 You can start a python server locally to see the genrated documentation
33 
34 `$ cd build/html; python -m http.server 5000`
35 
36 Navigate to `localhost:5000` on your browser to see the generated documentation.

38 Using a Debugger/Programmer
39 ------
40 
41 This repository supports two debuggers/programmers, namely [Black Magic Probe (BMP)](https://github.com/blacksphere/blackmagic/wiki) and Segger JLink.
42 
43 #### BMP ####
44 For getting started with the Black Magic Probe follow this [guide](https://github.com/blacksphere/blackmagic/wiki/Getting-Started).
45 \n On some Linux installations you may get permissions errors.
46 
47  /dev/ttyACM0: Permission denied
48 
49 This can be fixed by adding your user to the accessible group as follows (you may need to log out and back in for this to take effect):
50 
51 
52  $ stat /dev/ttyACM0 | grep Gid
53  Access: (0660/crw-rw----) Uid: ( 0/ root) Gid: ( 987/ dialout)
54 
55 
56 
57  Add user to the group, in our case `dailout`
58 
59 
60 
61  $ sudo adduser $USER dialout
62 
63 
64 \n Then, as mentioned [here](https://github.com/blacksphere/blackmagic/wiki/Frequently-Asked-Questions#its-annoying-to-look-up-an-always-changing-device-name-on-linux) create a file named `/etc/udev/rules.d/99-blackmagic.rules` with the following contents:
65 
66  # Black Magic Probe
67  # there are two connections, one for GDB and one for uart debugging
68  SUBSYSTEM=="tty", ATTRS{interface}=="Black Magic GDB Server", SYMLINK+="ttyBmpGdb"
69  SUBSYSTEM=="tty", ATTRS{interface}=="Black Magic UART Port", SYMLINK+="ttyBmpTarg"
70 
71 Then unplug / replug the probe, or restart the computer.
72 
73 #### Segger JLink ####
74 
75 - Download the SEGGER JLink software pack from [here](https://www.segger.com/downloads/jlink)
76 \n The latest version at the time of writing for a x64 Debian based environment is V6.32i and can be found [here](https://www.segger.com/downloads/jlink/JLink_Linux_V632i_x86_64.deb)
77 - Get the nrfjprog to work with the JLink. Its not in the repository as it does not have an open license.
78  - Download the nRF5x-Command-Line-Tools-Linux64 available at https://www.nordicsemi.com/eng/nordic/download_resource/51388/20/11220429
79  - Extract the "nrfjprog" folder in the "nrf5x-firmware > utils" folder in the main repository that you have cloned.
80 
81 Using the proprietary nRF5x SDK
82 ------
83 Get the components part of nRF5x SDK. Its not in the repository as it does not have an open license.
84 - Download the [nRF5_SDK_15.0.0_a53641a.zip](http://developer.nordicsemi.com/nRF5_SDK/nRF5_SDK_v15.x.x/nRF5_SDK_15.0.0_a53641a.zip) and extract it.
85 - Copy the "components" folder of the SDK to the "nrf5x-firmare" repository and rename it as "SDK_components"
86 - You might have to edit the Makefile of the application you want to compile, to set the `INCLUDEDIRS`, `C_SRC_DIRS` and `C_SRC` to the appropriate files and folders in the SDK.

88 Installing nrfutil
89 ------
90 nrfutil is a Python package and command-line utility that supports Device Firmware Updates (DFU) and cryptographic functionality.
91 
92 Follow these [instructions](https://infocenter.nordicsemi.com/index.jsp?topic=%2Fug_nrfutil%2FUG%2Fnrfutil%2Fnrfutil_installing.html) to install this utility.

94 Setting up IDE
95 ------
96 IDE(Integrated Development Environment) is useful for fast development of project. User can setup any IDE for project development.
97 \n User can also choose not to use any IDE at all. Two major IDEs which are used widely are:
98 
99 - NetBeans IDE
100 - Eclipse IDE
101 
102 \n It is advised to use the versions which comes with C/C++ plugins by default.
103 \n Follow the instructions given in following link to setup your IDE
104 
105 - NetBeans IDE :
106 
107  + [NetBeans Download](https://netbeans.org/downloads/) Select the one for C/C++.
108 
109  + [ARM ToolChain installation in NetBeans](http://www.dalbert.net/?p=239)
110 
111  + To get all the intellisense working perfectly without creating a mess, Download the [.netbeans.zip](https://github.com/Appiko/nrf5x-firmware/raw/development/doc/.netbeans.zip) file and import it using the import option under `tools > options`
112 
113  + Create a new Netbeans project and make sure the tool collection is `ARM(GNU)`, and select configuration mode is set to `Automatic`
114 
115  + On creating a successful project with intellisense, Netbeans would give a pop up saying *Code assistance successful*
116 
117 - Eclipse
118  + [Eclipse Download](https://www.eclipse.org/cdt/)
119  + [ARM ToolChain installation in Eclipse](https://gnu-mcu-eclipse.github.io/toolchain/arm/install/)
120 
121 
122 > NOTE : Use IDE only for development of application. While compiling and uploading of program use "make " utilities in terminal.
