3 # Project to test individual hardware module implemented on board
4 This board can be seen as many hardware modules implemented together.
5 This appliacton will test the each of those modules separately so that
6 manufacturer can check the board before passing it further on manufaturing cycle.
7 
8 These tests are designed to verify working of certain circuit part by comparing
9 certain critical value with the pre-calculated value.
10 
11 Before going through these tests please go through hardware discription for this
12 board. Otherwise it's difficult to understand these tests.
13 
14 These tests are as follow :
15 
16 * Power Supply Test
17 * DC/DC Converter Test
18 * LED Test
19 * Crystal Test
20 * RC Filter Test
21 * Low Pass Filter Test
22 * POT Test
23 
24 To do these tests user needs to follow the 'Board level Testing reference Manual'
25 Maths and logic behind every test is explained below.
26 
27 ## Power Supply Test :
28 This is the simplest test. If controller is able to boot and program is loaded
29 properly, It'll give positive status as return.
30 
31 If return value of this function is 1 then its successful.
32 
33 ## DC/DC Converter Test :
34 Internal DC/DC converter is used for regualating the DC supply given at battery
35 socket. If this device fails SensePi will keep rebooting.
36 If return value of this function is 1 then this test is successful
37 
38 ## LED Test :
39 This test is to see if LED is present and properly working or not. This is the
40 only test which is operator dependent. Here operator has to look at LED blinking.
41 
42 If first two test are successful then this test will always give positive result.
43 
44 
45 ## Crystal Test :
46 This test is to check if 32768 Hz Crystal is present and working or not. This
47 crystal is required to provide clock signal to all peripheral devices.
48 
49 In this test we first try to enable external crystal for low frequency clock.
50 If and after it gets enabled, it's frequency will be checked by comparing it
51 to high speed crystal clock present in SoC itself. For that profiler_timer module
52 is used which uses high speed crystal clock. It'll check for 1sec and compare
53 it for 1% error.
54 
55 If return value is 1 then crystal is present and error is less than 1%.
56 
57 ## RC Test :
58 This test is to check the RC circuit present before 'amp_offset' pin which is
59 supposed to get DC component of output from 2 stage amplifire. RC circuit is used
60 to filter the remaining AC component present in 'offset signal'. The desired
61 value of this offset value is (Vdd/2).
62 
63 In this test average offset value is calculated over 1.2s and then it is compared
64 with calculated offset value, which is (Vdd/2), for 5% error. If avg value is in
65 the 5% error margin then test was successful.
66 
67 ## Low Pass Filter Test :
68 This test is to check the filter circuit present before 'amp_signal' pin which
69 is treated as PIR signal. Low Pass Filter(LPF) has cut-off frequency of 1Hz.
70 
71 To check this test we need to provide signal with specifications such as :
72 
73 > V(p-p) : 1mv<br/>
74 > V(offset) : 0.9v<br/>
75 > Frequency : 10Hz<br/>
76 
77 In this test controller will check if it receives 10Hz signal at 'amp_signal'
78 pin or not.
79 
80 This freqiency will be calculated by zero crossing detection. In this method it
81 is checked for how many times does signal crosses a reference voltage around
82 which signal is oscillating in certain time duration, generally 1s, and then
83 divide that number by 2. For our application we are using ADC samples and
84 comparing with offset signal with 0.2% error to detect crossing.
85 
86 If this calculated frequency is equal to 10Hz, test is successful.
87 
88 ## POT Test :
89 This test is to check if POT is present and working. This POT value controls the
90 amplification of second stage amplifire.
91 
92 In this test, amp_signal's Vrms is calculated for two different POT values, 0
93 and 45, and compared if Vrms for POT value 45 is at least greater than double of
94 Vrms for POT value 0.
95 
96 If POT is working fine then this differece will be greater. Otherwise not.
